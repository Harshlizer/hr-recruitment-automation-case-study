# Publishing Checklist

Before publishing this case study to GitHub, verify the following:

- Remove company name, tenant name, and client identifiers
- Remove all candidate personal data and screenshots with visible PII
- Remove internal URLs, email addresses, Teams channels, and SharePoint paths
- Remove exported flow packages that contain environment-specific metadata
- Replace real field names when they reveal sensitive internal structure
- Use mock examples instead of production records
- Keep screenshots only if names, emails, phones, IDs, and tenant details are fully redacted
- Describe Adobe Sign integration at a high level if internal templates are confidential

## Recommended Optional Additions

- Add 1 to 3 redacted screenshots into an `assets/` folder
- Add a custom architecture diagram exported as PNG
- Add a short section with measurable impact if you can state it safely

## Good Portfolio Signals

- Clear business problem
- Cross-system automation
- Human approval loop
- Reporting visibility
- Privacy-aware documentation
